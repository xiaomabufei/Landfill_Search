Data Indicators
18 indicators per landfill — each core indicator includes a source reference (URL + type) for full traceability

Core Indicators 7 items · with source ref

Landfill Type
Classification as sanitary landfill (engineered, lined) or dump (uncontrolled). Critical for emission factor calculation in GHG inventories.
landfill_type → dump | sanitary landfill
Gas Collection
Whether the site has landfill gas (LFG) capture systems. Determines methane emission reduction potential.
has_gas_collection → yes | no
Technology Type
Flaring: burn-off only. Electrification: gas-to-electricity via engines. Purification: upgrade to biomethane for pipeline injection.
gas_collection_technology → flaring | electrification | purification
Collection Rate
Percentage of generated landfill gas that is actually captured. Typically 50-85% for well-managed sites. Key parameter for emission models.
gas_collection_rate → percentage
Start Year
Year the landfill began accepting waste. Used to calculate waste-in-place and estimate current gas generation phase.
start_year → year
Closure Year
Year the site stopped accepting waste, or null if still operational. Post-closure sites still generate gas for 20-30 years.
final_year → year | null
Gas System Start
Year gas collection equipment was installed. Used to determine how long emissions were uncontrolled before capture began.
gas_collection_start_year → year | null

Extended Indicators 11 items · collected when available

Capacity
Total engineered volume in cubic meters. Indicates site scale and remaining lifetime.
waste_capacity_m3
Annual Intake
Tons of waste received per year. Used for waste accumulation modeling.
annual_waste_intake_tons
Waste Types
Municipal solid waste, industrial, hazardous, inert, etc. Affects gas composition and generation rates.
waste_types_accepted
Area
Site footprint in hectares. Correlates with capacity and environmental impact zone.
area_hectares
Operator
Company or municipality managing the site. Useful for cross-referencing data sources.
operator
Depth
Fill depth in meters. Deeper sites generate gas longer and have different leachate characteristics.
depth_meters
Liner Type
HDPE, clay, composite, or none. Determines groundwater contamination risk.
liner_type
Leachate Treatment
How contaminated water is managed. On-site treatment, off-site disposal, or recirculation.
leachate_treatment
Status
Current operational phase: active, closed, or post-closure monitoring.
status → operational | closed | post-closure
Environmental Issues
Known contamination, violations, odor complaints, or legal proceedings.
environmental_issues
Methane Emissions
Estimated annual CH₄ release in tons. The primary GHG metric for landfill climate impact.
methane_emission_tons_year
Key Features
Built for reliability, traceability, and ease of use
🔍

Multi-Engine Search

Playwright automates Chrome/WebKit/Firefox to scrape Google Search, Scholar, and Maps. Bilingual keyword extraction (EN + target language).

✅

4D Validation

Format checks, logic validation (start ≤ end year), completeness tracking, and source verification. Errors highlighted in bold red.

🗺️

Interactive Maps

CartoDB + Leaflet.js map with color-coded markers. Click for details + source links. Chart.js statistics dashboard.

🔄

Checkpoint Resume

Interrupt anytime with Ctrl+C. Next run automatically resumes from where it stopped. Results saved after each landfill.

🖥️

Multi-Browser Parallel

Auto-detects installed browsers. Multiple workers with different fingerprints and staggered delays to avoid detection.

📋

Rich Logging

Timestamped logs with progress bar, ETA, collapsible details per landfill, and separate error log for quick triage.

3 Ways to Use
Choose the interface that fits your workflow
macOS App

Desktop GUI

Build as a native macOS app. Enter a landfill name or select a file, click search. Real-time progress in app window.

./build_app.sh
Web UI

Browser Interface

Flask web app at localhost:5000. Upload files, view real-time logs, download results. Works on any device in your network.

python web_app.py
Command Line

CLI / Script

Full control with flags: workers, range, engines, batch size. Best for automation and large-scale batch processing.

python main.py data.xlsx --scrape --headed
