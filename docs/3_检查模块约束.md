# 3. 检查模块约束

## 目标

对搜索结果进行自动化检查，确保数据准确性和可靠性。

---

## 检查维度

| 检查项 | 函数 | 说明 |
|--------|------|------|
| 格式校验 | `check_format()` | landfill_type 只能是 dump / sanitary landfill 等 |
| 逻辑校验 | `check_logic()` | start_year ≤ final_year，无 gas 则相关字段为 none |
| 完整性检查 | `check_completeness()` | 标记所有缺失字段 |
| 来源检查 | `check_refs()` | 有数据但无来源链接 |

## 异常等级

| 等级 | 标记 | 处理 |
|------|------|------|
| 🔴 错误 | `error` | 必须修正（格式不合规或逻辑矛盾） |
| 🟡 警告 | `warning` | 需人工确认（数据缺失或来源缺失） |
| 🟢 通过 | `pass` | 无需处理 |

## 运行方式

```bash
# 通过 Pipeline 自动运行
python main.py data/raw/ITA.xlsx

# 单独运行
python src/check/validator.py
```

输出：`output/{CODE}_check_report.md`
