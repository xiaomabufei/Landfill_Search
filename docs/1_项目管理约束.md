# 1. 项目管理约束

## 目标

对整个填埋场信息搜索项目进行统一管理，包括文档维护、版本控制和环境管理。

---

## 约束规则

### 1.1 文档管理

- 每个核心模块有独立的约束文档，记录功能、状态和变更。
- 项目根目录维护 `README.md`（用户入口）和 `docs/项目总览.md`（技术总览）。
- 文档更新应与代码修改 **同步进行**。

### 1.2 数据管理

| 目录 | 内容 | 说明 |
|------|------|------|
| `data/raw/` | 原始数据文件 | 支持 xlsx/csv/tsv/json |
| `output/` | 搜索结果 JSON + 检查报告 + HTML | 中间产物和最终输出 |
| `data/browser_state/` | 浏览器 cookies | 自动管理，减少验证码 |
| `logs/` | 运行日志 | Pipeline 日志 + 错误日志 |
| `output/html/` | HTML 可视化页面 | 自动生成 |

数据命名规范：`{国家代码}.xlsx` / `{国家代码}.json`（如 `ITA.xlsx`）。

### 1.3 环境管理

- `run_test.sh` 自动检测环境并安装缺失依赖。
- 依赖列表维护在 `requirements.txt`。
- Playwright Chromium 通过 `python -m playwright install chromium` 安装。
- 敏感信息（API Key 等）通过 `.env` + `.gitignore` 管理。

### 1.4 版本控制

- 使用 Git 进行版本管理。
- 提交信息格式：`[模块] 简要描述`。
- `.gitignore` 排除：`.env`、`__pycache__/`、`data/browser_state/`。
