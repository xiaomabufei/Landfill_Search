# CLAUDE.md — 项目协作约束

## 基本规则

1. **语言**：回答问题时尽量使用中文。
2. **简洁明了**：用简洁明了的语言回复；如果参考了资料或文献，尽量附上链接。
3. **项目规划优先**：在开始写代码之前，先构建项目的整体规划和介绍文档。如果还没有,先和用户一起商讨和分析目的，完善项目整体规划。商讨过程中你要主动的引导用户。
4. **修改代码流程**：
   - 先给出大致规划，分为细致的步骤让用户 review。
   - 用户确认后，再按步骤逐步修改。
   - 每一步修改都要有对应的检测代码（测试）。
5. **回复结尾**：每次回复完之后都要附上一个奋斗的表情包 💪🔥。
